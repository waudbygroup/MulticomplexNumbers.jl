User Guide

This guide provides a complete tour of MulticomplexNumbers.jl functionality.

The Multicomplex Type

The core type is Multicomplex{T,N,C} where:

  • T: The base scalar type (must be <:Real, e.g., Float64, BigFloat)
  • N: The order (number of imaginary units)
  • C: The number of components (always C = 2^N)
julia> using MulticomplexNumbers
julia> # Examine types
       z = 1.0 + 2.0*im11.0 + 2.0*im1
julia> typeof(z)Multicomplex{Float64, 1, 2}
julia> w = 1.0 + 2.0*im1 + 3.0*im2(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> typeof(w)Multicomplex{Float64, 2, 4}

Type Hierarchy

Number
└── Multicomplex{T,N,C}
    ├── Multicomplex{Float64,0,1}  # Just a real number
    ├── Multicomplex{Float64,1,2}  # Complex-like
    ├── Multicomplex{Float64,2,4}  # Bicomplex
    ├── Multicomplex{Float64,3,8}  # Tricomplex
    └── ...

Creating Multicomplex Numbers

Using Imaginary Units

The package exports im1 through im6:

julia> using MulticomplexNumbers
julia> im1  # First imaginary unit0 + 1*im1
julia> im2  # Second imaginary unit(0 + 0*im1) + (1 + 0*im1)*im2
julia> im3  # Third imaginary unit((0 + 0*im1) + (0 + 0*im1)*im2) + ((1 + 0*im1) + (0 + 0*im1)*im2)*im3
julia> # Build multicomplex numbers by combination
       z = 3.0 + 4.0*im1                           # Order 13.0 + 4.0*im1
julia> w = 1.0 + 2.0*im1 + 3.0*im2                 # Order 2 (auto-promoted)(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> v = 1.0 + im1 + im2 + im3 + im1*im2*im3    # Order 3((1.0 + 1.0*im1) + (1.0 + 0.0*im1)*im2) + ((1.0 + 0.0*im1) + (0.0 + 1.0*im1)*im2)*im3

Direct Constructors

julia> using MulticomplexNumbers
julia> # Order 0 (scalar)
       Multicomplex(5.0)5.0
julia> # Order 1 (from two reals)
       Multicomplex(1.0, 2.0)  # = 1 + 2*im11.0 + 2.0*im1
julia> # Order 1 (from Complex)
       Multicomplex(3.0 + 4.0im)  # = 3 + 4*im13.0 + 4.0*im1
julia> # Order 2 (from four reals): 1 + 2*im1 + 3*im2 + 4*im1*im2
       Multicomplex(1.0, 2.0, 3.0, 4.0)(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Order 2 (from two Complex numbers)
       Multicomplex(1.0 + 2.0im, 3.0 + 4.0im)  # = (1+2*im1) + (3+4*im1)*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2

Accessing Components

Real and Imaginary Parts

The real() and imag() functions return multicomplex numbers of order N-1:

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # For order 2: real/imag return order 1
       real(z)  # The "real" part with respect to im21.0 + 2.0*im1
julia> imag(z)  # The "imaginary" part with respect to im23.0 + 4.0*im1
julia> # Apply recursively
       real(real(z))  # Gets the truly real component1.0

Direct Component Access

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Get all components as a vector
       flat(z)4-element StaticArraysCore.SVector{4, Float64} with indices SOneTo(4):
 1.0
 2.0
 3.0
 4.0
julia> # Get a specific component (1-indexed)
       component(z, 1)  # Coefficient of 11.0
julia> component(z, 2)  # Coefficient of im12.0
julia> component(z, 3)  # Coefficient of im23.0
julia> component(z, 4)  # Coefficient of im1*im24.0
julia> # Get the first (most real) component
       realest(z)1.0

Component Ordering

Components follow a binary pattern based on which imaginary units are present:

OrderIndexBasis Element
111
12i₁
211
22i₁
23i₂
24i₁i₂
311
32i₁
33i₂
34i₁i₂
35i₃
36i₁i₃
37i₂i₃
38i₁i₂i₃

The pattern: index k has the units corresponding to bits set in k-1 (0-indexed).

Arithmetic Operations

Basic Arithmetic

julia> using MulticomplexNumbers
julia> a = 1.0 + 2.0*im1 + 3.0*im2(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> b = 2.0 - 1.0*im1 + 0.5*im2(2.0 - 1.0*im1) + (0.5 + 0.0*im1)*im2
julia> # Addition and subtraction
       a + b(3.0 + 1.0*im1) + (3.5 + 0.0*im1)*im2
julia> a - b(-1.0 + 3.0*im1) + (2.5 + 0.0*im1)*im2
julia> -a(-1.0 - 2.0*im1) + (-3.0 - 0.0*im1)*im2
julia> # Scalar operations
       3.0 * a(3.0 + 6.0*im1) + (9.0 + 0.0*im1)*im2
julia> a / 2.0(0.5 + 1.0*im1) + (1.5 + 0.0*im1)*im2
julia> a + 5.0(6.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2

Multiplication

Multicomplex multiplication uses the matrix representation internally:

julia> using MulticomplexNumbers
julia> a = 1.0 + 2.0*im11.0 + 2.0*im1
julia> b = 3.0 + 4.0*im13.0 + 4.0*im1
julia> a * b  # = (1*3 - 2*4) + (1*4 + 2*3)*im1 = -5 + 10*im1-5.0 + 10.0*im1
julia> # Verify the algebra
       im1 * im1  # = -1-1 + 0*im1
julia> im1 * im2  # = im1*im2 (distinct product)(0 + 0*im1) + (0 + 1*im1)*im2
julia> im2 * im1  # Commutative!(0 + 0*im1) + (0 + 1*im1)*im2

Division

Division works except when dividing by zero divisors:

julia> using MulticomplexNumbers
julia> a = 1.0 + 2.0*im11.0 + 2.0*im1
julia> b = 3.0 + 4.0*im13.0 + 4.0*im1
julia> a / b0.44 + 0.08*im1
julia> inv(b)  # = 1/bERROR: setindex!(::StaticArraysCore.SMatrix{2, 2, Float64, 4}, value, ::Int) is not defined.

Zero Divisors

For N ≥ 2, some non-zero multicomplex numbers cannot be inverted:

z = 1.0 + 1.0*im1*im2  # This has a zero divisor issue
# (1 + im1*im2)(1 - im1*im2) = 1 - (im1*im2)² = 1 - 1 = 0

Conjugation and Norms

Multicomplex Conjugation

Conjugation inverts the sign of the highest-order imaginary component:

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> conj(z)  # Inverts im2 component sign(1.0 + 2.0*im1) + (-3.0 - 4.0*im1)*im2
julia> # For order 1, this matches complex conjugation
       w = 1.0 + 2.0*im11.0 + 2.0*im1
julia> conj(w)1.0 - 2.0*im1

Absolute Values and Norms

julia> using MulticomplexNumbers
julia> using LinearAlgebra
julia> z = 3.0 + 4.0*im13.0 + 4.0*im1
julia> abs2(z)  # Sum of squares of components25.0
julia> abs(z)   # sqrt(abs2(z))5.0
julia> norm(z)  # Euclidean norm (same as abs for order 1)5.0

Type Conversions and Promotion

Converting to Multicomplex

julia> using MulticomplexNumbers
julia> # Reals promote to multicomplex
       z = 1.0 + 2.0*im11.0 + 2.0*im1
julia> z + 3.0  # 3.0 is promoted to Multicomplex4.0 + 2.0*im1
julia> # Complex promotes too
       z + (3.0 + 4.0im)4.0 + 6.0*im1
julia> # Different orders promote to the higher order
       a = 1.0 + im1        # Order 11.0 + 1.0*im1
julia> b = 2.0 + im2        # Order 2(2.0 + 0.0*im1) + (1.0 + 0.0*im1)*im2
julia> a + b                 # Result is order 2(3.0 + 1.0*im1) + (1.0 + 0.0*im1)*im2

Converting from Multicomplex

julia> using MulticomplexNumbers
julia> z = Multicomplex(5.0)  # Order 05.0
julia> # Convert to real if purely real
       Float64(z)5.0

FFT Support

When FFTW is loaded, in-place FFT is available:

using MulticomplexNumbers
using FFTW

# Create bicomplex data
data = [Multicomplex(rand(4)...) for _ in 1:64]

# FFT along im1 (treating im1 as the complex unit)
fft!(data, 1)

# FFT along im2
fft!(data, 2)

See the Tutorials for complete NMR examples.

Performance Tips

  1. Use Float64: It's the fastest for most applications

  2. Avoid order changes: Mixing orders triggers promotion, which allocates

  3. Pre-allocate arrays: For FFT operations, reuse arrays when possible

  4. Consider StaticArrays: The internal representation uses SVector, which is stack-allocated

  5. Profile high orders: Order N uses 2^N components. Order 6 has 64 components per number!

Common Patterns

Pattern 1: Numerical Differentiation

julia> using MulticomplexNumbers
julia> f(x) = sin(x) * cos(x)f (generic function with 1 method)
julia> x = 1.01.0
julia> h = 1e-1001.0e-100
julia> # First derivative
       f_prime = imag(f(x + h*im1)) / h-0.4161468365471423
julia> # Exact: f'(x) = cos(2x)
       cos(2x)-0.4161468365471424

Pattern 2: Processing 2D Data

using MulticomplexNumbers
using FFTW

# Create 2D bicomplex data
data = [Multicomplex(rand(4)...) for i in 1:64, j in 1:64]

# FFT both dimensions
fft!(data, 1)  # Direct
fft!(data, 2)  # Indirect

Pattern 3: Working with Components

julia> using MulticomplexNumbers
julia> # Extract the im1*im2 coefficient from a result
       z = (1.0 + im1 + im2)^2(-1.0 + 2.0*im1) + (2.0 + 2.0*im1)*im2
julia> component(z, 4)  # The im1*im2 term2.0
