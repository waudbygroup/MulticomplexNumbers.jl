Accessing Components

This page explains how to access and extract components from multicomplex numbers.

Real and Imaginary Parts

The real and imag functions return multicomplex numbers of order N-1:

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> real(z)         # The "real" part with respect to im21.0 + 2.0*im1
julia> imag(z)         # The "imaginary" part with respect to im23.0 + 4.0*im1
julia> real(real(z))   # Gets the truly real component1.0
julia> realest(z)      # Get the first (most real) component1.0

Direct Component Access

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> flat(z)          # Get all components as a vector4-element StaticArraysCore.SVector{4, Float64} with indices SOneTo(4):
 1.0
 2.0
 3.0
 4.0
julia> component(z, 1)  # Get a specific component (1-indexed)1.0
julia> component(z, 2)  # Coefficient of im12.0
julia> component(z, 3)  # Coefficient of im23.0
julia> component(z, 4)  # Coefficient of im1*im24.0

Component Ordering

Components follow a binary pattern based on which imaginary units are present:

OrderIndexBasis Element
111
12i₁
211
22i₁
23i₂
24i₁i₂
311
32i₁
33i₂
34i₁i₂
35i₃
36i₁i₃
37i₂i₃
38i₁i₂i₃

The pattern: index k has the units corresponding to bits set in k-1 (0-indexed).

Order and Properties

julia> using MulticomplexNumbers
julia> order(1.0 + im1)        # Get the order1
julia> order(1.0 + im1 + im2)2
julia> isreal(5.0 + 0.0*im1)   # Check propertiestrue
julia> isreal(1.0 + 2.0*im1)false

Matrix Representations

Every multicomplex number can be represented as a real matrix:

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im11.0 + 2.0*im1
julia> M = matrep(z)2×2 StaticArraysCore.SMatrix{2, 2, Float64, 4} with indices SOneTo(2)×SOneTo(2):
 1.0  -2.0
 2.0   1.0
julia> size(M)     # Matrix properties(2, 2)
julia> M[:, 1]     # The first column contains the components2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 1.0
 2.0
julia> flat(z)2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 1.0
 2.0
julia> w = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> size(matrep(w))     # Bicomplex gives 4×4(4, 4)

The matrix representation preserves algebra:

julia> w = 3.0 + 4.0*im13.0 + 4.0*im1
julia> flat(z * w)                     # Direct multiplication2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 -5.0
 10.0
julia> (matrep(z) * matrep(w))[:, 1]   # Multiplication via matrices2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 -5.0
 10.0

Complex Array Views

The ascomplex function views multicomplex arrays as complex arrays. This is particularly useful for extracting specific complex sub-components in NMR applications.

julia> using MulticomplexNumbers
julia> data = [Multicomplex(1.0, 2.0, 3.0, 4.0) for i in 1:3]  # Create a bicomplex array3-element Vector{Multicomplex{Float64, 2, 4}}:
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> c1 = ascomplex(data, 1)  # View as complex along im1 (extracts real + im1*i components)2×3 reinterpret(reshape, ComplexF64, ::Vector{Multicomplex{Float64, 2, 4}}) with eltype ComplexF64:
 1.0+2.0im  1.0+2.0im  1.0+2.0im
 3.0+4.0im  3.0+4.0im  3.0+4.0im
julia> size(c1)  # (2, 3) - the 2 is from 2^(N-1)(2, 3)
julia> c2 = ascomplex(data, 2)  # View as complex along im2 (extracts real + im2*i components)2×3 reinterpret(reshape, ComplexF64, ::Array{Float64, 3}) with eltype ComplexF64:
 1.0+3.0im  1.0+3.0im  1.0+3.0im
 2.0+4.0im  2.0+4.0im  2.0+4.0im
julia> size(c2)(2, 3)
julia> c_default = ascomplex(data)  # Default is highest order, same as c22×3 reinterpret(reshape, ComplexF64, ::Array{Float64, 3}) with eltype ComplexF64:
 1.0+3.0im  1.0+3.0im  1.0+3.0im
 2.0+4.0im  2.0+4.0im  2.0+4.0im
