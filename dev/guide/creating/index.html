Creating Multicomplex Numbers

This page explains how to create and work with multicomplex numbers.

The Multicomplex Type

The core type is Multicomplex{T,N,C} where:

  • T: The base scalar type (must be <:Real, e.g., Float64, BigFloat)
  • N: The order (number of imaginary units)
  • C: The number of components (always C = 2^N)
julia> using MulticomplexNumbers
julia> # Examine types
       z = 1.0 + 2.0*im11.0 + 2.0*im1
julia> typeof(z)Multicomplex{Float64, 1, 2}
julia> w = 1.0 + 2.0*im1 + 3.0*im2(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> typeof(w)Multicomplex{Float64, 2, 4}

Type Hierarchy

Number
└── Multicomplex{T,N,C}
    ├── Multicomplex{Float64,0,1}  # Just a real number
    ├── Multicomplex{Float64,1,2}  # Complex-like
    ├── Multicomplex{Float64,2,4}  # Bicomplex
    ├── Multicomplex{Float64,3,8}  # Tricomplex
    └── ...

Using Imaginary Units

The package exports im1 through im6:

julia> using MulticomplexNumbers
julia> im1  # First imaginary unit0 + 1*im1
julia> im2  # Second imaginary unit(0 + 0*im1) + (1 + 0*im1)*im2
julia> im3  # Third imaginary unit((0 + 0*im1) + (0 + 0*im1)*im2) + ((1 + 0*im1) + (0 + 0*im1)*im2)*im3
julia> # Build multicomplex numbers by combination
       z = 3.0 + 4.0*im1                           # Order 13.0 + 4.0*im1
julia> w = 1.0 + 2.0*im1 + 3.0*im2                 # Order 2 (auto-promoted)(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> v = 1.0 + im1 + im2 + im3 + im1*im2*im3    # Order 3((1.0 + 1.0*im1) + (1.0 + 0.0*im1)*im2) + ((1.0 + 0.0*im1) + (0.0 + 1.0*im1)*im2)*im3

Direct Constructors

julia> using MulticomplexNumbers
julia> # Order 0 (scalar)
       Multicomplex(5.0)5.0
julia> # Order 1 (from two reals)
       Multicomplex(1.0, 2.0)  # = 1 + 2*im11.0 + 2.0*im1
julia> # Order 1 (from Complex)
       Multicomplex(3.0 + 4.0im)  # = 3 + 4*im13.0 + 4.0*im1
julia> # Order 2 (from four reals): 1 + 2*im1 + 3*im2 + 4*im1*im2
       Multicomplex(1.0, 2.0, 3.0, 4.0)(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Order 2 (from two Complex numbers)
       Multicomplex(1.0 + 2.0im, 3.0 + 4.0im)  # = (1+2*im1) + (3+4*im1)*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2

Type Conversions and Promotion

Converting to Multicomplex

julia> using MulticomplexNumbers
julia> # Reals promote to multicomplex
       z = 1.0 + 2.0*im11.0 + 2.0*im1
julia> z + 3.0  # 3.0 is promoted to Multicomplex4.0 + 2.0*im1
julia> # Complex promotes too
       z + (3.0 + 4.0im)4.0 + 6.0*im1
julia> # Different orders promote to the higher order
       a = 1.0 + im1        # Order 11.0 + 1.0*im1
julia> b = 2.0 + im2        # Order 2(2.0 + 0.0*im1) + (1.0 + 0.0*im1)*im2
julia> a + b                 # Result is order 2(3.0 + 1.0*im1) + (1.0 + 0.0*im1)*im2

Converting from Multicomplex

julia> using MulticomplexNumbers
julia> z = Multicomplex(5.0)  # Order 05.0
julia> # Convert to real if purely real
       Float64(z)5.0

Working with Higher Precision

MulticomplexNumbers supports any Real scalar type:

julia> using MulticomplexNumbers
julia> # BigFloat for high precision
       z = big"1.0" + big"2.0"*im11.0 + 2.0*im1
julia> typeof(z)Multicomplex{BigFloat, 1, 2}
