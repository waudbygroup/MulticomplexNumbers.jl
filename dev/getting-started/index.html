Getting Started

This guide will help you get up and running with MulticomplexNumbers.jl quickly.

Installation

MulticomplexNumbers.jl is a registered Julia package. Install it using Julia's package manager:

using Pkg
Pkg.add("MulticomplexNumbers")

Or in the package REPL (press ]):

pkg> add MulticomplexNumbers

Optional: FFT Support

For Fourier transform capabilities, also install FFTW:

Pkg.add("FFTW")

Basic Usage

julia> using MulticomplexNumbers

Creating Multicomplex Numbers

The package provides imaginary units im1 through im6 for constructing multicomplex numbers:

julia> # Order 1: Complex numbers (2 components)
       z1 = 1.0 + 2.0*im11.0 + 2.0*im1
julia> # Order 2: Bicomplex numbers (4 components)
       z2 = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Order 3: Tricomplex numbers (8 components)
       z3 = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im3((1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2) + ((4.0 + 0.0*im1) + (0.0 + 0.0*im1)*im2)*im3

You can also use direct constructors:

julia> # From real components
       Multicomplex(1.0, 2.0)           # = 1 + 2*im11.0 + 2.0*im1
julia> Multicomplex(1.0, 2.0, 3.0, 4.0) # = 1 + 2*im1 + 3*im2 + 4*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # From Complex numbers
       Multicomplex(1.0 + 2.0im)        # = 1 + 2*im11.0 + 2.0*im1

Key Properties

Each imaginary unit squares to -1:

julia> im1 * im1-1 + 0*im1
julia> im2 * im2(-1 + 0*im1) + (0 + 0*im1)*im2

Unlike quaternions, multicomplex units commute:

julia> im1 * im2 == im2 * im1true

However, the product of different units is hyperbolic (squares to +1):

julia> (im1 * im2)^2(1 + 0*im1) + (0 + 0*im1)*im2

Arithmetic Operations

All standard operations are supported:

julia> a = 1.0 + 2.0*im1 + 3.0*im2(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> b = 2.0 + 1.0*im1 - 1.0*im2(2.0 + 1.0*im1) + (-1.0 + 0.0*im1)*im2
julia> # Addition and subtraction
       a + b(3.0 + 3.0*im1) + (2.0 + 0.0*im1)*im2
julia> a - b(-1.0 + 1.0*im1) + (4.0 + 0.0*im1)*im2
julia> # Multiplication and division
       a * b(3.0 + 5.0*im1) + (5.0 + 1.0*im1)*im2
julia> a / b(0.25 + 1.0*im1) + (1.5 - 0.25*im1)*im2
julia> # Powers and transcendental functions
       a^2(-12.0 + 4.0*im1) + (6.0 + 12.0*im1)*im2
julia> exp(a)(1.119883852040778 - 2.4469908589319616*im1) + (-0.1596355717531187 + 0.34881008787510875*im1)*im2
julia> log(a)(0.9878109296453564 + 0.294001301773784*im1) + (1.079399465171232 - 0.6412373393653843*im1)*im2
julia> sqrt(a)(1.4224843356318504 + 0.4882605175879972*im1) + (0.9433503781502243 - 0.32380022216404086*im1)*im2

Extracting Components

julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Real and imaginary parts (returns multicomplex of order N-1)
       real(z)1.0 + 2.0*im1
julia> imag(z)3.0 + 4.0*im1
julia> # Get all components as a vector
       flat(z)4-element StaticArraysCore.SVector{4, Float64} with indices SOneTo(4):
 1.0
 2.0
 3.0
 4.0
julia> # Get the "most real" component (first component)
       realest(z)1.0
julia> # Get a specific component by index
       component(z, 2)  # The im1 coefficient2.0

Quick Example: Numerical Differentiation

The most powerful application of multicomplex numbers is computing derivatives with machine precision:

julia> # Compute the derivative of f(x) = sin(x) at x = π/4
       f(x) = sin(x)f (generic function with 1 method)
julia> # Use an extremely small step size (impossible with finite differences!)
       h = 1e-1001.0e-100
julia> x = π/4 + h*im10.7853981633974483 + 1.0e-100*im1
julia> # Evaluate the function
       result = f(x)0.7071067811865474 + 7.071067811865474e-101*im1
julia> # Extract the derivative from the imaginary part
       derivative = imag(result) / h0.7071067811865474
julia> # Compare to exact value: cos(π/4) = √2/2 ≈ 0.7071...
       cos(π/4)0.7071067811865476

The result matches the exact derivative to machine precision!
