Getting Started

This guide will help you get up and running with MulticomplexNumbers.jl quickly.

Installation

MulticomplexNumbers.jl is a registered Julia package. Install it using Julia's package manager:

using Pkg
Pkg.add("MulticomplexNumbers")

Or in the package REPL (press ]):

pkg> add MulticomplexNumbers

Optional: FFT Support

For Fourier transform capabilities, also install FFTW:

Pkg.add("FFTW")

Basic Usage

julia> using MulticomplexNumbers

Creating Multicomplex Numbers

The package provides imaginary units im1 through im6 for constructing multicomplex numbers:

julia> z1 = 1.0 + 2.0*im1  # Order 1: Complex numbers (2 components)1.0 + 2.0*im1
julia> z2 = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2  # Order 2: Bicomplex numbers (4 components)(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> z3 = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im3  # Order 3: Tricomplex numbers (8 components)((1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2) + ((4.0 + 0.0*im1) + (0.0 + 0.0*im1)*im2)*im3

You can also use direct constructors:

julia> Multicomplex(1.0, 2.0)           # From real components1.0 + 2.0*im1
julia> Multicomplex(1.0, 2.0, 3.0, 4.0)(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> Multicomplex(1.0 + 2.0im)        # From complex numbers1.0 + 2.0*im1

Key Properties

Each imaginary unit squares to -1:

julia> im1 * im1-1 + 0*im1
julia> im2 * im2(-1 + 0*im1) + (0 + 0*im1)*im2

Unlike quaternions, multicomplex units commute:

julia> im1 * im2 == im2 * im1true

However, the product of different units is hyperbolic (squares to +1):

julia> (im1 * im2)^2(1 + 0*im1) + (0 + 0*im1)*im2

Arithmetic Operations

All standard operations are supported:

julia> a = 1.0 + 2.0*im1 + 3.0*im2(1.0 + 2.0*im1) + (3.0 + 0.0*im1)*im2
julia> b = 2.0 + 1.0*im1 - 1.0*im2(2.0 + 1.0*im1) + (-1.0 + 0.0*im1)*im2
julia> a + b     # Addition(3.0 + 3.0*im1) + (2.0 + 0.0*im1)*im2
julia> a - b     # Subtraction(-1.0 + 1.0*im1) + (4.0 + 0.0*im1)*im2
julia> a * b     # Multiplication(3.0 + 5.0*im1) + (5.0 + 1.0*im1)*im2
julia> a / b     # Division(0.25 + 1.0*im1) + (1.5 - 0.25*im1)*im2
julia> a^2       # Powers(-12.0 + 4.0*im1) + (6.0 + 12.0*im1)*im2
julia> sqrt(a)   # Square root(1.4224843356318504 + 0.4882605175879972*im1) + (0.9433503781502243 - 0.32380022216404086*im1)*im2
julia> exp(a)    # Exponentials(1.1198838520407761 - 2.4469908589319607*im1) + (-0.15963557175311865 + 0.3488100878751088*im1)*im2
julia> log(a)    # Logarithms(0.9878109296453564 + 0.294001301773784*im1) + (1.079399465171232 - 0.6412373393653843*im1)*im2

Extracting Components

julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> real(z)   # Real / imaginary parts (returns multicomplex of order N-1)1.0 + 2.0*im1
julia> imag(z)3.0 + 4.0*im1
julia> flat(z)   # Get all components as a vector4-element StaticArraysCore.SVector{4, Float64} with indices SOneTo(4):
 1.0
 2.0
 3.0
 4.0
julia> realest(z)  # Get the "most real" component (first component)1.0
julia> component(z, 2)  # Get a specific component by index2.0

Quick Example: Numerical Differentiation

One application of multicomplex numbers is computing derivatives with machine precision:

julia> f(x) = sin(x)   # Compute the derivative of f(x) = sin(x) at x = π/4f (generic function with 1 method)
julia> h = 1e-100      # Use an extremely small step size1.0e-100
julia> x = π/4 + h*im10.7853981633974483 + 1.0e-100*im1
julia> result = f(x)   # Evaluate the function0.7071067811865475 + 7.071067811865476e-101*im1
julia> derivative = imag(result) / h   # Extract the derivative from the imaginary part0.7071067811865476
julia> cos(π/4)    # Compare to exact value: cos(π/4) = √2/2 ≈ 0.7071...0.7071067811865476

The result matches the exact derivative to machine precision!
