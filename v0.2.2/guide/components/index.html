Accessing Components

This page explains how to access and extract components from multicomplex numbers.

Real and Imaginary Parts

The real and imag functions return multicomplex numbers of order N-1:

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # For order 2: real/imag return order 1
       real(z)  # The "real" part with respect to im21.0 + 2.0*im1
julia> imag(z)  # The "imaginary" part with respect to im23.0 + 4.0*im1
julia> # Apply recursively
       real(real(z))  # Gets the truly real component1.0

Direct Component Access

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Get all components as a vector
       flat(z)4-element StaticArraysCore.SVector{4, Float64} with indices SOneTo(4):
 1.0
 2.0
 3.0
 4.0
julia> # Get a specific component (1-indexed)
       component(z, 1)  # Coefficient of 11.0
julia> component(z, 2)  # Coefficient of im12.0
julia> component(z, 3)  # Coefficient of im23.0
julia> component(z, 4)  # Coefficient of im1*im24.0
julia> # Get the first (most real) component
       realest(z)1.0

Component Ordering

Components follow a binary pattern based on which imaginary units are present:

OrderIndexBasis Element
111
12i₁
211
22i₁
23i₂
24i₁i₂
311
32i₁
33i₂
34i₁i₂
35i₃
36i₁i₃
37i₂i₃
38i₁i₂i₃

The pattern: index k has the units corresponding to bits set in k-1 (0-indexed).

Order and Properties

julia> using MulticomplexNumbers
julia> # Get the order
       order(1.0 + im1)1
julia> order(1.0 + im1 + im2)2
julia> # Check properties
       isreal(5.0 + 0.0*im1)true
julia> isreal(1.0 + 2.0*im1)false

Matrix Representations

Every multicomplex number can be represented as a real matrix:

julia> using MulticomplexNumbers
julia> z = 1.0 + 2.0*im11.0 + 2.0*im1
julia> M = matrep(z)2×2 StaticArraysCore.SMatrix{2, 2, Float64, 4} with indices SOneTo(2)×SOneTo(2):
 1.0  -2.0
 2.0   1.0
julia> # Matrix properties
       size(M)(2, 2)
julia> # The first column contains the components
       M[:, 1]2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 1.0
 2.0
julia> flat(z)2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 1.0
 2.0
julia> # Bicomplex gives 4×4
       w = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> size(matrep(w))(4, 4)

The matrix representation preserves algebra:

julia> # Multiplication via matrices
       w = 3.0 + 4.0*im13.0 + 4.0*im1
julia> flat(z * w)2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 -5.0
 10.0
julia> (matrep(z) * matrep(w))[:, 1]2-element StaticArraysCore.SVector{2, Float64} with indices SOneTo(2):
 -5.0
 10.0

Complex Array Views

The ascomplex function views multicomplex arrays as complex arrays. This is particularly useful for extracting specific complex sub-components in NMR applications.

julia> using MulticomplexNumbers
julia> # Create a bicomplex array
       data = [Multicomplex(1.0, 2.0, 3.0, 4.0) for i in 1:3]3-element Vector{Multicomplex{Float64, 2, 4}}:
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # View as complex along im1 (extracts real + im1*i components)
       c1 = ascomplex(data, 1)2×3 reinterpret(reshape, ComplexF64, ::Vector{Multicomplex{Float64, 2, 4}}) with eltype ComplexF64:
 1.0+2.0im  1.0+2.0im  1.0+2.0im
 3.0+4.0im  3.0+4.0im  3.0+4.0im
julia> size(c1)  # (2, 3) - the 2 is from 2^(N-1)(2, 3)
julia> # View as complex along im2 (extracts real + im2*i components)
       c2 = ascomplex(data, 2)2×3 reinterpret(reshape, ComplexF64, ::Array{Float64, 3}) with eltype ComplexF64:
 1.0+3.0im  1.0+3.0im  1.0+3.0im
 2.0+4.0im  2.0+4.0im  2.0+4.0im
julia> size(c2)(2, 3)
julia> # Default is highest order
       c_default = ascomplex(data)  # Same as ascomplex(data, 2)2×3 reinterpret(reshape, ComplexF64, ::Array{Float64, 3}) with eltype ComplexF64:
 1.0+3.0im  1.0+3.0im  1.0+3.0im
 2.0+4.0im  2.0+4.0im  2.0+4.0im

Extracting Specific Complex Components

For NMR data, you often want to extract specific complex sub-spectra:

julia> using MulticomplexNumbers
julia> # Create bicomplex data
       z = 1.0 + 2.0*im1 + 3.0*im2 + 4.0*im1*im2(1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> # Extract the real/im1 complex component
       # This gives you a Complex number from the "real" part (with respect to im2)
       real_im1_component = real(z)  # Returns 1.0 + 2.0*im11.0 + 2.0*im1
julia> # Extract the real/im2 complex component
       # Re-arrange to view im2 as the complex unit
       # For a single number, you can manually extract:
       real_part = realest(z)  # 1.01.0
julia> im2_part = component(z, 3)  # 3.03.0
julia> # Complex(real_part, im2_part) would give 1.0 + 3.0im
       
       # For arrays, ascomplex does this automatically
       data = [Multicomplex(1.0, 2.0, 3.0, 4.0) for _ in 1:4]4-element Vector{Multicomplex{Float64, 2, 4}}:
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
 (1.0 + 2.0*im1) + (3.0 + 4.0*im1)*im2
julia> complex_view_im1 = ascomplex(data, 1)  # View as complex along im12×4 reinterpret(reshape, ComplexF64, ::Vector{Multicomplex{Float64, 2, 4}}) with eltype ComplexF64:
 1.0+2.0im  1.0+2.0im  1.0+2.0im  1.0+2.0im
 3.0+4.0im  3.0+4.0im  3.0+4.0im  3.0+4.0im
julia> complex_view_im2 = ascomplex(data, 2)  # View as complex along im22×4 reinterpret(reshape, ComplexF64, ::Array{Float64, 3}) with eltype ComplexF64:
 1.0+3.0im  1.0+3.0im  1.0+3.0im  1.0+3.0im
 2.0+4.0im  2.0+4.0im  2.0+4.0im  2.0+4.0im

Performance Tips

  1. Use flat() once: If you need multiple components, call flat() once and index the result

  2. Avoid order changes: Mixing orders triggers promotion, which allocates

  3. Pre-allocate arrays: For FFT operations, reuse arrays when possible

  4. Consider StaticArrays: The internal representation uses SVector, which is stack-allocated

  5. Profile high orders: Order N uses 2^N components. Order 6 has 64 components per number!
